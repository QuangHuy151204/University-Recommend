5. Phần AI nên làm thế nào để vừa đủ sức, vừa dễ bảo vệ

Đây là phần dễ bị “tham”. Bạn nên chọn hướng vừa đủ thông minh, dễ demo, dễ giải thích.

Cách làm AI phù hợp cho đồ án

Có 3 mức:

Mức 1: Rule-based recommendation

Hệ thống hỏi:

em thích ngành gì?

điểm dự kiến bao nhiêu?

thích học khu vực nào?

tài chính thế nào?

Sau đó dùng luật để recommend.

Ví dụ:

điểm 20–22, thích CNTT, học Hà Nội, học phí trung bình → gợi ý nhóm trường A

điểm 24+, thích kinh tế → gợi ý nhóm trường B

Ưu điểm:

dễ làm

dễ giải thích

dễ bảo vệ

Mức 2: Chatbot AI + dữ liệu nội bộ

Dùng LLM để trả lời bằng ngôn ngữ tự nhiên, nhưng chatbot chỉ dựa trên dữ liệu website.

Ví dụ:

bạn lưu thông tin trường/ngành/điểm chuẩn vào database

chatbot đọc dữ liệu này

trả lời theo ngữ cảnh

Cách này đẹp hơn mức 1 rất nhiều.

Mức 3: Recommendation + AI chatbot

Kết hợp:

recommendation engine để gợi ý trường/ngành

chatbot để trò chuyện và giải thích vì sao gợi ý như vậy

Đây là hướng đẹp nhất cho đồ án.

Lời khuyên thực tế

Với đồ án tốt nghiệp, bạn nên làm:

core system ổn

recommendation logic rõ ràng

chatbot AI ở mức hỗ trợ hỏi đáp + giải thích gợi ý

Tức là:

recommendation là trái tim

chatbot là lớp giao tiếp thông minh

Như vậy vừa chắc, vừa có điểm mới.

## Đã có (ổn cho demo / bảo vệ)

- Auth đầy đủ: đăng ký, verify email, login, forgot/reset, admin login, JWT cookie
- CRUD master data: trường, ngành, liên kết trường–ngành, điểm chuẩn, PT xét tuyển

- Admin dashboard tabbed: stats + CRUD 5 nhóm dữ liệu
- Tích hợp API qua `lib/api.ts` + services

- Phạm vi Hà Nội, điểm chuẩn 2023–2025

## Còn thiếu / chưa đủ (so với spec)

### Chức năng người dùng
| Mục | Trạng thái |
|-----|------------|
| Trang chủ: ngành nổi bật | Thiếu |
| Chatbot nhanh embed trên home | Chỉ link, chưa widget |
| So sánh **ngành** | Thiếu hoàn toàn |
| Lịch sử gợi ý (`GET /recommendations/my`) | API có, **UI chưa dùng** |
| Tra cứu **không cần login** | Hiện `AuthGate` chặn hầu hết trang — lệch spec “tra cứu công khai” |
| Cơ hội việc làm / block career riêng | Chỉ mô tả ngành nếu có trong DB |

### Admin
| Mục | Trạng thái |
|-----|------------|
| Quản lý user | API có (`listUsersAdmin`), **không có panel** |
| FAQ / tri thức chatbot | Thiếu |
| Xem/review chat history | Thiếu |
| Import Excel từ UI | Chỉ CLI `npm run import:excel` |

### Chatbot / AI
| Mục | Trạng thái |
|-----|------------|
| Học bổng (`ask_scholarship`) | Không có DB → trả text hướng dẫn |
| Cơ sở vật chất/KTX (`ask_facilities`) | Thiếu dữ liệu chi tiết |
| `intent.txt` corpus | Có `intent-corpus.ts` + script generate; **chưa chắc đồng bộ 100%** với `intent.txt` |
| `engine: "rule"` nhiều | **Đúng thiết kế** (skip rewrite để giữ số liệu DB), không phải Ollama tắt |

### Kỹ thuật / chất lượng
| Mục | Trạng thái |
|-----|------------|
| `POST /recommendations` lưu lịch sử khi đã login | **Bug nhỏ**: thiếu `OptionalJwtAuthGuard` → `req.user` luôn null dù có cookie |
| E2E HTTP tests (auth, CRUD, chatbot API) | Gần như không có; `test:e2e` còn stale |
| Secrets trong git history | `.env` từng push — cần rotate |
| Production deploy | Chưa có Docker/CI, chưa audit bảo mật đầy đủ |
| Một số ghi chú `Yêu cầu.txt` (dòng 690–693) | **Lỗi thời**: compare chatbot đã sửa; JWT đã cookie; intent coverage đã cải thiện |

### Dữ liệu
- Không phủ toàn quốc (cố ý — Hà Nội)
- Một số ngành/khối thiếu cutoff → gợi ý trả `[]` (cần UX message rõ hơn)
- Ngân sách backend vẫn enum low/medium/high
